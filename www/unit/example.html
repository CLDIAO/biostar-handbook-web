A web page that may be a plain html, may have markdown (see below) and can be accessed as a django template.

The django template can access data loaded from a special file in the root folder called context.py. The content of this module will be available as a template variable called context. Anything is now possible ;-). The current context.py for example loads the table of contents and helps to generate the navigation bar on top.

For example the function context.say_hello will be rendered as: {{ context.say_hello }}

Or it can read the context.numbers: {{ context.numbers }} in a for loop like so: {% for value in context.numbers %} {{ value }} mississippi, {% endfor %}

Restart the server if you alter context.py or if you add a new file. Pyblue will reload changes in existing files but won't pick up new files or a changed context.py
