A pyblue web page may be a simple, plain html page, it may contain markdown markup and it can be processed as a django template.

When processed as a django template it can access data loaded from a special file in the root folder called context.py. The content of this module will be available as a template variable called context. This allows users to add any type of data into every template. The current context.py for example loads the table of contents and helps to generate the navigation bar on top.

For example the function context.say_hello will be rendered as: {{ context.say_hello }}

Or it can read the context.NUMBERS: {{ context.NUMBERS }} in a for loop like so: 

{% verbatim %}
{% for value in context.NUMBERS %}
    {{ value }} Mississippi,
{% endfor %}
{% endverbatim %}
to produce: {% for value in context.NUMBERS %} {{ value }} Mississippi, {% endfor %} See the Django Template Language for what is possible to achieve with templating.

Restart the server if you alter context.py or if you add a new file. Pyblue will reload changes in existing files but won't pick up new files or a changed context.py

Pyblue supports code embedding from other files:

{% verbatim %} {% code "helloworld.sh" %} {% endverbatim %}

Pyblue is smart when linking and will find the correct location just by the name, so you can move objects around withouth breaking links. When the file is included the code contained in it gets syntax highlighted:

{% code "helloworld.sh" %} {% endblock %}
